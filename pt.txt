    script = [
        "from opentrons import protocol_api",
        "metadata = {'apiLevel': '2.14'}",
        "def run(protocol: protocol_api.ProtocolContext):",
        "    # Load hardware",
        "    plate = protocol.load_labware('corning_96_wellplate_360ul_flat', 1)",
        "    tiprack = protocol.load_labware('opentrons_96_tiprack_20ul', 2)",
        "    p20 = protocol.load_instrument('p20_single_gen2', 'left', tip_racks=[tiprack])",
        "    # Execute Optimized Actions"
    ]
   
    for action in optimized_actions:
        if action.action_type == "pick_up_tip":
            script.append("    p20.pick_up_tip()")
        elif action.action_type == "aspirate":
            script.append(f"    p20.aspirate({action.volume_ul}, {action.source_location})")

        elif action.action_type == "drop_tip":

# Do not implement full academic IRL overnight

Gemini’s “Bayesian MCMC IRL from 5 flawless scripts” idea sounds impressive, but it is risky and probably overkill.

A true maximum-entropy IRL system over a full lab MDP is too large for a hackathon unless heavily simplified. IRL is generally the problem of inferring a reward function from observed behavior, and the literature notes challenges around inference accuracy, generalization, sensitivity to priors, and growing complexity with problem size. ([ScienceDirect][7])

For the hackathon, build this instead:

## Use inverse preference reward learning

Define trajectory features:

```text
phi(trajectory) =
[
  contamination_violations,
  pipette_range_violations,
  well_capacity_violations,
  missing_tip_events,
  unknown_location_events,
  ungrounded_reagents,
  simulator_errors,
  unnecessary_tip_changes,
  transfer_count,
  mix_completeness,
  cold_chain_violations,
  total_robot_motion_proxy
]
```

Learn a linear reward:

```text
R(trajectory) = w · phi(trajectory)
```

Train weights using expert-vs-corrupted pairs:

```text
P(expert > corrupted) = sigmoid(w · (phi(expert) - phi(corrupted)))
```

This is enough to truthfully say:

## Layer 6: Policy repair optimizer

This is the “fixer.”

It should apply deterministic repairs first:

| Error                        | Repair                                      |
| ---------------------------- | ------------------------------------------- |
| Missing tip                  | Insert `pick_up_tip()`                      |
| Tip contamination            | Insert `drop_tip()` and `pick_up_tip()`     |
| Volume too large for pipette | Switch pipette or split transfer            |
| Well overflow                | Split across wells or flag impossible       |
| Unknown reagent location     | Ask for human input or use default deck map |
| Missing mix                  | Insert `mix()` after dispense               |
| Simulator error              | Send back to verifier/compiler loop         |

Do not rely on the LLM to fix everything. Use rules. Judges will trust deterministic repair much more than another LLM call.

## `verifier.py`

Implement hard checks:

```python
def verify_ir(ir_actions, state):
    violations = []

    for action in ir_actions:
        if action.op == "Aspirate" and not state.tip_attached[action.pipette]:
            violations.append({"type": "NO_TIP", "action": action})

        if action.op == "Aspirate" and not pipette_can_handle(action.pipette, action.volume_ul):
            violations.append({"type": "PIPETTE_RANGE", "action": action})

        if action.op == "Dispense" and would_overflow(action.destination, action.volume_ul):
            violations.append({"type": "WELL_OVERFLOW", "action": action})

        if action.op == "Aspirate" and state.tip_reagent[action.pipette] not in [None, action.reagent]:
            violations.append({"type": "CROSS_CONTAMINATION", "action": action})

    return violations
```

# How to do “full Bayesian IRL” without faking it

Use **Bayesian Maximum Entropy IRL** over a finite lab-protocol MDP.

Your model:

```text
State s:
  deck layout
  liquid volumes per well
  reagent class per well
  current pipette
  tip attached?
  tip contamination class
  current location
  module temperature state
  completed semantic goals

Action a:
  pick_up_tip
  drop_tip
  aspirate(volume, source)
  dispense(volume, destination)
  transfer(volume, source, destination, tip_policy)
  distribute(volume, source, destinations)
  mix(repetitions, volume, location)
  blow_out
  touch_tip
  delay
  set_temperature

Reward:
  R_w(s, a) = w^T phi(s, a)
```

Posterior:

```text
p(w | D) ∝ p(w) ∏ᵢ p(τᵢ | w)

p(τᵢ | w) = exp(Σ_t wᵀ φ(s_t, a_t)) / Z_w(Tᵢ)
```

For hackathon feasibility, make `Z_w(Tᵢ)` exact over a **bounded candidate set** generated by your protocol grammar:

```text
candidate trajectories for same task =
  expert trajectory
  same task with different tip policies
  same task with different pipette choices
  same task with different transfer ordering
  same task with missing mix
  same task with unsafe contamination variants
  same task with inefficient but safe variants
```

This is not the same as heuristic reward scoring. The “corrupted” or alternative trajectories are not labels; they are part of the denominator / partition set. Your posterior still learns `w` by explaining why the expert trajectory is more probable than other feasible trajectories.

Use:

```text
NumPyro or PyMC
NUTS / HMC sampler
Gaussian prior over weights
feature-standardized trajectories
4 chains
posterior mean + credible intervals
posterior predictive score for generated protocols
```

For the demo, show:

```text
posterior mean reward weights
95% credible intervals
expert trajectory likelihood
unsafe trajectory likelihood
generated trajectory likelihood
```

That makes it look mathematically serious.

class LabState:
    def _init_(self):
        self.tip_attached = {"p20": False, "p300": False}
        self.tip_reagent = {"p20": None, "p300": None}
        self.current_volume = {"p20": 0, "p300": 0}
        self.well_volumes = {}  # well_id → current volume
        self.loaded_labware = {}
        self.loaded_instruments = {}

def verify_ir(ir_ops):
    state = LabState()
    violations = []
    
    for i, action in enumerate(ir_ops):
        if action.op == "LoadLabware":
            state.loaded_labware[action.alias] = action
        
        elif action.op == "LoadInstrument":
            state.loaded_instruments[action.name] = action
        
        elif action.op == "PickUpTip":
            pipette = action.pipette
            state.tip_attached[pipette] = True
            state.tip_reagent[pipette] = None
            state.current_volume[pipette] = 0
        
        elif action.op == "DropTip":
            pipette = action.pipette
            if state.current_volume[pipette] > 0:
                violations.append({
                    "type": "DROP_TIP_WITH_LIQUID",
                    "severity": "CRITICAL",
                    "action_idx": i,
                    "message": f"Dropping tip while containing {state.current_volume[pipette]} µL"
                })
            state.tip_attached[pipette] = False
            state.tip_reagent[pipette] = None
        
        elif action.op == "Aspirate":
            pipette = action.pipette
            
            # Check: Tip attached?
            if not state.tip_attached[pipette]:
                violations.append({
                    "type": "ASPIRATE_NO_TIP",
                    "severity": "CRITICAL",
                    "action_idx": i,
                    "message": f"{pipette} aspirating without attached tip"
                })
            
            # Check: Pipette range?
            instr = state.loaded_instruments[pipette]
            if action.volume_ul < instr.min_volume or action.volume_ul > instr.max_volume:
                violations.append({
                    "type": "PIPETTE_RANGE_VIOLATION",
                    "severity": "CRITICAL",
                    "action_idx": i,
                    "message": f"{pipette} range is {instr.min_volume}–{instr.max_volume} µL, attempted {action.volume_ul} µL"
                })
            
            # Check: Source exists?
            if action.source not in state.loaded_labware:
                violations.append({
                    "type": "UNKNOWN_SOURCE",
                    "severity": "CRITICAL",
                    "action_idx": i,
                    "message": f"Source {action.source} not loaded"
                })
            
            state.current_volume[pipette] = action.volume_ul
            state.tip_reagent[pipette] = action.reagent
        
        elif action.op == "Dispense":
            pipette = action.pipette
            
            # Check: Tip attached?
            if not state.tip_attached[pipette]:
                violations.append({
                    "type": "DISPENSE_NO_TIP",
                    "severity": "CRITICAL",
                    "action_idx": i
                })
            
            # Check: Destination exists?
            if action.destination not in state.loaded_labware:
                violations.append({
                    "type": "UNKNOWN_DESTINATION",
                    "severity": "CRITICAL",
                    "action_idx": i
                })
            
            # Check: Well overflow?
            well_id = action.destination
            dest_labware = state.loaded_labware.get(well_id.split("/")[0])
            if dest_labware:
                current = state.well_volumes.get(well_id, 0)
                if current + action.volume_ul > dest_labware.max_volume_ul:
                    violations.append({
                        "type": "WELL_OVERFLOW",
                        "severity": "CRITICAL",
                        "action_idx": i,
                        "message": f"Well {well_id} capacity {dest_labware.max_volume_ul} µL, would have {current + action.volume_ul} µL"
                    })
            
            state.well_volumes[well_id] = current + action.volume_ul
            state.current_volume[pipette] -= action.volume_ul
    
    return violations

def extract_trajectory_features(ir_ops, violations):
    features = {
        "contamination_violations": 0,
        "pipette_range_violations": 0,
        "well_overflow_violations": 0,
        "aspirate_no_tip_violations": 0,
        "unknown_location_violations": 0,
        "drop_tip_with_liquid_violations": 0,
        "total_violations": len(violations),
        
        # Efficiency
        "tip_changes": sum(1 for op in ir_ops if op.op == "DropTip"),
        "transfer_count": sum(1 for op in ir_ops if op.op == "Aspirate"),
        "mix_events": sum(1 for op in ir_ops if op.op == "Mix"),
        
        # Safety bonuses
        "tip_changed_between_different_reagents": 0,
        "complete_transfer_pairs": 0,
    }
    
    # Count safety violations
    for v in violations:
        if v["type"] == "CROSS_CONTAMINATION":
            features["contamination_violations"] += 1
        # ... etc
    
    return features

def score_trajectory(features, reward_weights):
    score = 0
    for feature_name, weight in reward_weights.items():
        if feature_name in features:
            score += features[feature_name] * weight
    return score

def repair_ir(ir_ops, violations):
    if not violations:
        return ir_ops
    
    for violation in violations:
        action_idx = violation["action_idx"]
        
        if violation["type"] == "ASPIRATE_NO_TIP":
            # Insert pick_up_tip before this aspirate
            ir_ops.insert(action_idx, IROp(op="PickUpTip", pipette=ir_ops[action_idx].pipette))
        
        elif violation["type"] == "CROSS_CONTAMINATION":
            # Insert drop_tip and pick_up_tip before this aspirate
            pipette = ir_ops[action_idx].pipette
            ir_ops.insert(action_idx, IROp(op="DropTip", pipette=pipette))
            ir_ops.insert(action_idx + 1, IROp(op="PickUpTip", pipette=pipette))
        
        elif violation["type"] == "PIPETTE_RANGE_VIOLATION":
            # Switch to appropriate pipette (p20 ↔ p300)
            volume = ir_ops[action_idx].volume_ul
            if volume > 20:
                ir_ops[action_idx].pipette = "p300"
            else:
                ir_ops[action_idx].pipette = "p20"
        
        elif violation["type"] == "WELL_OVERFLOW":
            # Flag for human intervention (too complex to repair automatically)
            violations.append({
                "type": "HUMAN_ESCALATION",
                "message": f"Well {ir_ops[action_idx].destination} would overflow. Manual design required."
            })
        
        elif violation["type"] == "UNKNOWN_LOCATION":
            violations.append({
                "type": "HUMAN_ESCALATION",
                "message": f"Unknown source/destination {ir_ops[action_idx].source or ir_ops[action_idx].destination}. Clarification needed."
            })
    
    return ir_ops

def compile_to_opentrons(ir_ops):
    script = [
        "from opentrons import protocol_api",
        "",
        "metadata = {",
        '    "apiLevel": "2.14",',
        '    "protocolName": "ProtocolIR Generated Protocol"',
        "}",
        "",
        "def run(protocol: protocol_api.ProtocolContext):",
    ]
    
    labware_aliases = {}
    instrument_aliases = {}
    
    for op in ir_ops:
        if op.op == "LoadLabware":
            script.append(f"    {op.alias} = protocol.load_labware('{op.name}', {op.slot})")
            labware_aliases[op.alias] = op
        
        elif op.op == "LoadInstrument":
            tiprack_str = ", ".join([f"{alias}" for alias in op.tipracks])
            script.append(f"    {op.name} = protocol.load_instrument('{op.name}', '{op.mount}', tip_racks=[{tiprack_str}])")
            instrument_aliases[op.name] = op
        
        elif op.op == "PickUpTip":
            script.append(f"    {op.pipette}.pick_up_tip()")
        
        elif op.op == "DropTip":
            script.append(f"    {op.pipette}.drop_tip()")
        
        elif op.op == "Aspirate":
            # Parse source: "template_rack/A1" → template_rack["A1"]
            well = op.source.split("/")[1]
            labware = op.source.split("/")[0]
            script.append(f"    {op.pipette}.aspirate({op.volume_ul}, {labware}['{well}'])")
        
        elif op.op == "Dispense":
            well = op.destination.split("/")[1]
            labware = op.destination.split("/")[0]
            script.append(f"    {op.pipette}.dispense({op.volume_ul}, {labware}['{well}'])")
        
        elif op.op == "Mix":
            well = op.location.split("/")[1]
            labware = op.location.split("/")[0]
            script.append(f"    {op.pipette}.mix({op.repetitions}, {op.volume_ul}, {labware}['{well}'])")
    
    return "\n".join(script)

def simulate_opentrons(script_path):
    try:
        result = subprocess.run(
            ["python", "-m", "opentrons.simulate", script_path],
            capture_output=True,
            text=True,
            timeout=30
        )
        
        if result.returncode == 0:
            return {
                "passed": True,
                "log": result.stdout,
                "errors": []
            }
        else:
            return {
                "passed": False,
                "log": result.stdout,
                "errors": result.stderr.split("\n")
            }
    
    except subprocess.TimeoutExpired:
        return {
            "passed": False,
            "errors": ["Simulation timed out (>30s)"]
        }
    except Exception as e:
        return {
            "passed": False,
            "errors": [str(e)]
        }

def process_protocol(raw_text, reward_weights):
    # Step 1: Parse
    parsed = parse_protocol(raw_text)
    
    # Step 2: Ground
    grounded = ground_to_deck(parsed)
    
    # Step 3: IR
    ir = build_ir(grounded)
    
    # Step 4: Verify
    violations = verify_ir(ir)
    if violations:
        # Step 5: Repair
        ir = repair_ir(ir, violations)
        violations = verify_ir(ir)  # Re-check
    
    # Step 6: Score
    features = extract_features(ir, violations)
    score = score_trajectory(features, reward_weights)
    
    # Step 7: Compile
    script = compile_opentrons(ir)
    
    # Step 8: Simulate
    sim_result = simulate_opentrons(script)
    
    # Step 9: Report
    report = generate_audit_report(raw_text, parsed, ir, violations, [], score, score, sim_result)
    
    return {
        "script": script,
        "report": report,
        "violations": violations,
        "score": score,
        "simulation": sim_result
    }

# ProtocolIR: Reward-Guided Protocol Compiler for Safe Lab Automation

Runnable code is not safe code. LLM lab copilots generate Opentrons scripts that pass simulation but violate biological safety constraints (cross-contamination, pipette range, well capacity). *ProtocolIR* learns a reward model from expert robot demonstrations, automatically detects semantic safety violations, and repairs protocols before compilation.

## The Problem

- Standard LLMs can write syntactically valid Opentrons Python
- Scripts may still violate safety (reusing contaminated tips, ignoring pipette range, well overflow)
- No system today validates semantic lab safety, only whether code runs

## Our Solution

1. *Semantic Parser* — Convert messy protocols.io text to structured actions
2. *Typed IR* — Machine-readable intermediate representation
3. *Hard Verifier* — Enforce physical invariants (no tip, range, overflow)
4. *Reward Model* — Learn preferences from expert vs. corrupted trajectories
5. *Deterministic Repair* — Fix violations automatically
6. *Opentrons Compiler* — Generate verified Python code
7. *Simulator Validation* — Prove execution safety
8. *Audit Report* — Explain all changes to end user

Now, let's lock in the final, end-to-end architecture for **ProtocolIR**. This is not a simple LLM wrapper; it is a full-blown reward-guided compiler. We are going to separate this into two distinct pipelines: the **Offline Bayesian IRL Training** and the **Live LangGraph Inference Pipeline**.

Here is the exact step-by-step blueprint.

---

### Part 1: The Offline Bayesian IRL Training (The Brains)
Before your agent processes a single user prompt, it needs to learn the underlying mathematical rules of safe chemistry. Since you are bringing heavy computational statistics into this, we are doing true Bayesian Maximum Entropy IRL.

1.  **Define the Lab MDP:** You formally define the State $S$ (deck layout, well volumes, tip attachment status, contamination class) and Actions $A$ (pick up tip, aspirate, dispense, drop tip, mix).
2.  **Generate Counterfactuals:** Take your pristine Opentrons expert scripts (Dataset B) and programmatically generate corrupted versions (e.g., delete a `drop_tip` command, exceed pipette volume).
3.  **Sample the Posterior:** You define a linear reward function $R(s,a) = w^T \phi(s,a)$ based on extracted features (contamination events, unnecessary movements, tip changes). Using an MCMC sampler (like NUTS via PyMC or NumPyro), you sample the posterior distribution of the reward weights $p(w \mid D) \propto p(w) \prod_i p(\tau_i \mid w)$. The sampler finds the specific weights that make the expert trajectories mathematically highly probable and the corrupted ones highly improbable.
4.  **Export the Weights:** You extract the posterior mean reward weights and save them locally (e.g., `learned_weights.json`). 

---

### Part 2: The Live LangGraph Compiler Pipeline
This is the real-time pipeline that runs when a user inputs a messy protocol during your demo. It operates as a directed graph.

**Layer 1: Semantic Parser (The LLM Extractor)**
* **Input:** Messy text from protocols.io.
* **Action:** You pass the text to Codex (or your designated LLM) using strict `Pydantic` schemas. The LLM extracts the goal, materials, and a sequential list of semantic actions (e.g., "transfer", "mix", "temperature"). It also flags ambiguities. 

**Layer 2: Grounding Engine (The Spatial Mapper)**
* **Action:** A deterministic Python script takes the abstract semantic actions and maps them to physical Opentrons deck slots and labware (e.g., mapping "master mix" to `opentrons_24_tuberack_nest_1.5ml_snapcap` on Slot 3).

**Layer 3: Typed Intermediate Representation (IR)**
* **Action:** The grounded actions are converted into a strict, machine-readable JSON format (`IROp`). This looks like assembly code for the lab (e.g., `{"op": "Aspirate", "pipette": "p20", "volume_ul": 10, "source": "plate/A1"}`).

**Layer 4: Hard Safety Verifier (The Physics Engine)**
* **Action:** The IR is passed through a deterministic state tracker. It simulates the physical invariants. If a tip isn't attached before aspiration, or if a dispense action exceeds a well's maximum capacity, it flags a "Hard Violation."

**Layer 5: Learned Reward Scorer (The IRL Judge)**
* **Action:** The system loads your `learned_weights.json` (from Part 1) and scores the IR trajectory. It calculates how safe and efficient the sequence is based on your Bayesian posterior. 

**Layer 6: The Repair Policy (The Fixer)**
* **Action:** This is the LangGraph loop. If Layer 4 finds violations or Layer 5 returns a terrible reward score, the Repair Engine deterministically injects fixes into the IR. For example, if a cross-contamination violation was flagged, it injects a `DropTip` and `PickUpTip` operation. It loops back to Layer 4 until the sequence is mathematically safe.

**Layer 7: Deterministic Compiler**
* **Action:** Once the IR passes all safety checks and maximizes the reward, this layer translates the JSON operations directly into raw Opentrons Python API syntax. No LLMs are used here.

**Layer 8: Simulator Validation**
* **Action:** Your pipeline automatically executes `python -m opentrons.simulate output.py` in the background. It captures the stdout log to guarantee the script executes flawlessly without crashing the virtual robot.

**Layer 9: Audit Report Generation**
* **Output:** The system generates a final markdown report for the user. It shows the initial messy text, lists the specific safety violations the base LLM missed, details the deterministic repairs injected by the IRL engine, shows the massive jump in the mathematical reward score, and outputs the "PASS" from the Opentrons simulator.
